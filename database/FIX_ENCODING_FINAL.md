## المشكلة

## الحل النهائي

### الخطوة 1: تشغيل SQL لإصلاح البيانات

افتح PowerShell وانتقل إلى مجلد المشروع:

```powershell
cd "d:\My Documents\DIT Tech\موقع الجمعية ابو بسام\project"
```

ثم شغّل:

```powershell
psql -U postgres -d association_db -f admin-cms\database\fix_homepage_encoding.sql
```

هذا سيقوم بـ:
- حذف البيانات القديمة (التي تم إدخالها بترميز خاطئ)
- إعادة إدخال البيانات بترميز UTF-8 صحيح

### الخطوة 2: إعادة تشغيل CMS

1. أوقف CMS إذا كان يعمل (Ctrl+C)
2. أعد تشغيله:
```powershell
cd admin-cms
npm run dev
```

### الخطوة 3: التحقق

1. افتح CMS: `http://localhost:3001`
2. اذهب إلى "الصفحة الرئيسية"
3. تحقق من أن النصوص العربية تظهر بشكل صحيح

## ما تم إصلاحه

✅ تحديث اتصال قاعدة البيانات لاستخدام UTF-8 بشكل إجباري
✅ جميع API routes ترسل البيانات بترميز UTF-8
✅ البيانات في قاعدة البيانات تم إعادة إدخالها بترميز صحيح

## إذا استمرت المشكلة

1. تأكد من أن قاعدة البيانات تستخدم UTF-8:
```sql
SHOW server_encoding;  -- يجب أن يكون UTF8
```

2. إذا لم يكن UTF8، قم بتشغيل:
```sql
ALTER DATABASE association_db SET client_encoding = 'UTF8';
```

3. أعد تشغيل PostgreSQL إذا لزم الأمر
