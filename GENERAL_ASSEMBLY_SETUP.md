# إعداد صفحة الجمعية العمومية

## نظرة عامة

تم إنشاء نظام شامل لإدارة صفحة **الجمعية العمومية** التي تحتوي على:
- **معلومات عن الجمعية العمومية** (Hero section ونص تمهيدي)
- **أعضاء الجمعية العمومية** (من جدول `association_members` الموجود)
- **محاضر الاجتماعات** (ملفات PDF)

## الخطوات

### 1. إنشاء الجداول في قاعدة البيانات

قم بتشغيل ملف SQL التالي:

```powershell
psql -U postgres -d association_db -f admin-cms/database/create_general_assembly_tables.sql
```

هذا سينشئ الجداول التالية:
- `general_assembly_content` - محتوى الصفحة (Hero section ونص تمهيدي)
- `assembly_meetings` - اجتماعات الجمعية العمومية (مع ملفات PDF)

**ملاحظة:** جدول `association_members` موجود مسبقاً ويستخدم لعرض أعضاء الجمعية العمومية.

### 2. الوصول إلى صفحة الإدارة

1. سجّل الدخول إلى CMS: `http://localhost:3001`
2. من القائمة الجانبية، اختر **"الجمعية العمومية"**
3. ستجد تبويبات:
   - **محتوى الصفحة** - لإدارة Hero section والنص التمهيدي
   - **محاضر الاجتماعات** - لإدارة اجتماعات الجمعية (CRUD)

### 3. إدارة المحتوى

#### محتوى الصفحة
- **Hero Section:**
  - العنوان الرئيسي
  - العنوان الفرعي
  - الحالة (نشط/غير نشط)
- **النص التمهيدي:**
  - النص التمهيدي
  - الحالة (نشط/غير نشط)

#### محاضر الاجتماعات
- يمكنك إضافة/تعديل/حذف اجتماعات الجمعية
- كل اجتماع يحتوي على:
  - العنوان
  - الوصف (اختياري)
  - تاريخ الاجتماع
  - رقم الاجتماع (اختياري)
  - ملف المحضر (PDF)
  - اسم الملف (يتم ملؤه تلقائياً)
  - الترتيب
  - الحالة (نشط/غير نشط)

### 4. إدارة الأعضاء

أعضاء الجمعية العمومية يتم جلبهم من جدول `association_members` الموجود. يمكنك إدارتهم من:
- **أعضاء الجمعية** في CMS

## الملفات المحدثة/المضافة

### قاعدة البيانات
- `admin-cms/database/create_general_assembly_tables.sql` - جداول الصفحة

### API Routes
- `admin-cms/app/api/general-assembly-page/route.ts` - API لإدارة محتوى الصفحة
- `admin-cms/app/api/assembly-meetings/route.ts` - API لإدارة الاجتماعات (GET, POST)
- `admin-cms/app/api/assembly-meetings/[id]/route.ts` - API لإدارة اجتماع واحد (GET, PUT, DELETE)
- `admin-cms/app/api/public/general-assembly/route.ts` - Public API

### CMS Pages
- `admin-cms/app/general-assembly/page.tsx` - صفحة إدارة الجمعية العمومية
- `admin-cms/app/general-assembly/meetings/new/page.tsx` - صفحة إضافة اجتماع جديد
- `admin-cms/app/general-assembly/meetings/[id]/page.tsx` - صفحة تعديل اجتماع

### Frontend
- `admin-cms/lib/apiClient.ts` - إضافة وظائف API
- `src/lib/api.ts` - إضافة `getGeneralAssembly()`
- `src/pages/about/GeneralAssembly.tsx` - تحديث صفحة الواجهة الأمامية

### Navigation
- `admin-cms/components/AdminLayout.tsx` - إضافة رابط في القائمة الجانبية

## استخدام الصفحة

بعد إتمام الإعداد، يمكن الوصول إلى صفحة الجمعية العمومية من الموقع العام عبر:
- الرابط: `/about-assembly` (حسب التوجيه في `App.tsx`)

الصفحة تعرض تلقائياً:
- Hero section مع العنوان والعنوان الفرعي
- النص التمهيدي
- رئيس الجمعية (أول عضو أو عضو مع `category = 'chairman'`)
- باقي أعضاء الجمعية في شبكة
- محاضر الاجتماعات مع روابط التحميل

## ملاحظات مهمة

1. **الأعضاء:** يتم جلبهم من جدول `association_members` الموجود. تأكد من وجود بيانات الأعضاء في هذا الجدول.
2. **الاجتماعات:** يمكن إدارتها من صفحة "الجمعية العمومية" → "محاضر الاجتماعات"
3. **الملفات:** استخدم `ImageInput` component لرفع ملفات PDF. اسم الملف يتم ملؤه تلقائياً من رابط الملف.
4. **الترتيب:** يمكن ترتيب الاجتماعات باستخدام `order_index` و `meeting_date`.
